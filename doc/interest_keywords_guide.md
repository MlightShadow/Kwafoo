# 兴趣关键字配置指南

## 概述

Kwafoo新闻聚合系统支持配置用户感兴趣的关键字，AI评分器会分析新闻中是否命中这些关键字，并根据命中程度和相关性进行评分。

## 配置位置

配置文件：`config.toml`
配置节：`[ai.scoring]`
配置项：`interest_keywords`

## 配置方式

### 1. 空列表（默认）- 根据内容质量评分

```toml
[ai.scoring]
interest_keywords = []  # 空列表表示对所有新闻都感兴趣
```

**效果：**
- 系统会对所有新闻进行评分
- AI主要根据新闻内容质量和价值进行评分
- 适合想要全面了解各类新闻的用户

### 2. 指定兴趣关键字 - 只对包含关键字的新闻感兴趣

```toml
[ai.scoring]
# 只关注AI相关的关键字
interest_keywords = ["人工智能", "大模型", "ChatGPT", "机器学习", "深度学习"]
```

**效果：**
- AI会分析新闻中是否命中这些关键字
- 根据关键字的命中程度和相关性进行评分
- 适合专注于特定主题的用户

### 3. 混合配置 - 关注多个主题

```toml
[ai.scoring]
# 关注科技和金融关键字
interest_keywords = ["人工智能", "ChatGPT", "股票", "投资", "财报", "区块链"]
```

## AI评分逻辑

### 有兴趣关键字配置

AI会执行以下分析：

1. **关键字检测**
   - 分析新闻标题、摘要和内容
   - 找出所有命中的兴趣关键字
   - 记录每个关键字的命中位置

2. **命中程度分析**
   - 标题命中：权重最高
   - 摘要命中：权重中等
   - 内容命中：权重较低

3. **相关性评估**
   - 评估关键字与新闻主题的相关性
   - 判断关键字是否为核心话题
   - 考虑关键字的命中频率

4. **综合评分**
   - 根据命中程度和相关性给出0-100的评分
   - 提供详细的命中分析说明

**评分标准：**
- 90-100分：标题命中多个关键字，或命中关键字且与新闻主题高度相关
- 70-90分：标题命中1-2个关键字，或摘要命中多个关键字
- 60-70分：摘要命中关键字，或内容中多次命中关键字
- 40-60分：内容中命中少量关键字，或命中程度较弱
- 0-40分：没有命中任何关键字，或命中程度非常弱

### 无兴趣关键字配置

AI会根据新闻内容质量进行评分：

**评分标准：**
- 90-100分：新闻内容重要、有价值、信息量大、时效性强
- 70-90分：新闻内容有价值、信息量适中、有一定时效性
- 60-70分：新闻内容一般、信息量一般、时效性一般
- 40-60分：新闻内容较少、信息量有限、时效性较弱
- 0-40分：新闻内容质量差、信息量少、时效性差

## 关键字选择建议

### 1. 选择核心关键字
- 选择最能代表你兴趣的核心词汇
- 避免过于宽泛的关键字
- 优先选择专业术语和品牌名称

### 2. 考虑同义词
- 包含相关概念的不同表述
- 例如：AI、人工智能、机器学习、深度学习
- 提高命中率

### 3. 关注特定领域
- 专注于你真正感兴趣的领域
- 避免配置过多不相关的关键字
- 提高评分准确性

### 4. 定期优化
- 根据实际使用情况调整关键字
- 移除命中率低的关键字
- 添加新发现的相关关键字

## 注意事项

1. **关键字数量**
   - 建议配置5-15个关键字
   - 太少可能错过相关新闻
   - 太多可能导致评分不够精确

2. **关键字质量**
   - 选择具体、明确的关键字
   - 避免过于通用的词汇
   - 优先选择专业术语

3. **大小写敏感**
   - AI会自动处理大小写
   - 不需要区分大小写配置
   - "ChatGPT" 和 "chatgpt" 效果相同

4. **中英文混合**
   - 可以同时配置中英文关键字
   - 例如："人工智能", "AI", "ChatGPT"
   - 提高覆盖范围

5. **动态调整**
   - 可以根据实际使用情况调整关键字
   - 建议定期回顾和优化配置
   - 可以结合新闻阅读习惯进行调整

## 故障排除

### 问题1：配置后没有生效

**检查：**
1. 是否重启了服务器
2. 配置文件语法是否正确
3. 日志中是否显示配置加载信息

**解决：**
```bash
# 验证配置语法
python -c "import toml; print(toml.load(open('config.toml')))"

# 检查日志
tail -f logs/kwafoo.log | grep "兴趣关键字"
```

### 问题2：评分结果不符合预期

**检查：**
1. 关键字是否正确配置
2. 新闻中是否真的包含这些关键字
3. AI模型是否正常工作

**解决：**
- 查看AI评分日志中的命中详情
- 检查关键字的命中分析
- 调整关键字配置

### 问题3：某些新闻总是低分

**可能原因：**
1. 新闻中没有命中任何关键字
2. 命中的关键字与主题关联度低
3. 关键字配置不够全面

**解决：**
- 扩大关键字范围
- 添加相关概念的关键字
- 考虑使用空列表配置

## 最佳实践

1. **从少量关键字开始**
   - 先配置3-5个最感兴趣的关键字
   - 观察评分效果
   - 根据需要逐步添加

2. **定期回顾**
   - 每周检查一次评分结果
   - 查看命中详情分析
   - 根据阅读习惯调整

3. **结合其他配置**
   - 配合评分权重使用
   - 考虑重要程度描述
   - 综合调整各项参数

4. **监控日志**
   - 关注关键字命中情况
   - 查看AI评分分析
   - 及时发现配置问题

5. **灵活调整**
   - 根据不同时期调整关键字
   - 例如：财报季关注财务关键字
   - 技术发布会关注产品关键字

## 相关配置

兴趣关键字配置与其他评分配置协同工作：

### 评分权重配置

```toml
[ai.scoring]
topic_relevance_weight = 0.4    # 关联度权重
importance_weight = 0.4          # 重要程度权重
source_weight = 0.2              # 来源分权重
```

### 重要程度描述配置

```toml
[ai.scoring]
importance_description = """
判断新闻重要程度的标准：
1. 影响范围：全球性事件 > 国家级事件 > 地区性事件 > 局部事件
2. 影响程度：重大政策变化 > 重大技术突破 > 重要商业事件 > 一般新闻
3. 涉及人数：影响数百万人的事件 > 影响数十万人的事件 > 影响数千人的事件
"""
```

## 总结

兴趣关键字配置是Kwafoo系统个性化的重要功能，通过合理配置可以：

1. **提高相关性** - 只看到包含感兴趣关键字的新闻
2. **精准评分** - AI详细分析关键字命中情况
3. **节省时间** - 减少不相关新闻的干扰
4. **提升体验** - 获得更精准的新闻推荐

建议根据个人兴趣和使用习惯，灵活调整兴趣关键字配置，获得最佳的使用体验。通过AI的智能分析，系统会准确识别新闻中的关键字命中情况，并提供详细的评分理由。
